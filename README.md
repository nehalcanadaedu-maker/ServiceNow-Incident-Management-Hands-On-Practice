# ServiceNow-Incident-Management-Hands-On-Practice
This project demonstrates practical experience with incident creation and handling using a personal instance of ServiceNow. It simulates real-world IT support workflows such as ticket creation, prioritization, resolution, and documentation.
Here’s a clean **GitHub README.md** you can use for your ServiceNow practice project. I’ll structure it so you can just paste it and replace screenshot links.

## 📌 Project Overview

In this lab, I practiced the full **incident lifecycle management process**, including:

* Creating incidents in ServiceNow
* Assigning priority based on impact and urgency
* Updating ticket status (New → In Progress → Resolved)
* Adding work notes and resolution details
* Simulating real IT support scenarios

---

## 🧪 Key Skills Practiced

* Incident Management (ITSM fundamentals)
* Ticket lifecycle handling
* Priority & urgency mapping
* Work notes & documentation
* Basic IT support troubleshooting simulation

## 🔄 Workflow Summary

1. Incident created with user issue description
2. Priority set based on impact & urgency
3. Ticket moved to **In Progress**
4. Work notes added during investigation
5. Issue resolved and ticket closed
